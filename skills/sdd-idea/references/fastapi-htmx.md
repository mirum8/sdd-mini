## When to pick this stack

Pick this for a small web tool without an admin or auth: a calculator, a form with a result page, a single-page list, a microservice with a UI. Django ships with a lot (admin, migrations, ORM), but for a single screen it's too much. FastAPI is light, explicit, fast to start, with great typing. htmx covers interactivity without a JS framework. SQLAlchemy 2.0 is a transparent ORM with honest SQL.

## When NOT to pick

If the project needs an admin, auth, or user-facing form editing — prefer `django-htmx`: same htmx + server but with batteries included.

## Minimal MVP tech

- Python 3.12
- `fastapi>=0.115`
- `uvicorn[standard]>=0.30`
- `jinja2>=3.1` — templates

## Phase 1 recipe (handoff)

SDD does not automate the FastAPI scaffold — Claude builds Phase 1 from this recipe.

**What Phase 1 must produce:**

1. The files below.
2. `git init` if needed, `git add .gitignore`, `git commit --allow-empty -m "init"`.
3. `docker compose up -d --build`.
4. Initialize Alembic: `docker compose exec -T app alembic init alembic`, configure `env.py` to read `Base.metadata` from `app.models`.
5. UI gate for `app/templates/home.html` — invoke `frontend-design:frontend-design`.

### `Dockerfile`

```dockerfile
FROM python:3.12-slim

ENV PYTHONDONTWRITEBYTECODE=1 PYTHONUNBUFFERED=1 PIP_NO_CACHE_DIR=1

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends build-essential && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "5000", "--reload"]
```

### `app/main.py`

```python
from fastapi import FastAPI, Request
from fastapi.responses import HTMLResponse
from fastapi.templating import Jinja2Templates

PROJECT_NAME = "<Project display name>"

app = FastAPI(title=PROJECT_NAME)
templates = Jinja2Templates(directory="app/templates")


@app.get("/", response_class=HTMLResponse)
async def home(request: Request):
    return templates.TemplateResponse(
        "home.html", {"request": request, "project_name": PROJECT_NAME}
    )


@app.get("/health")
async def health():
    return {"status": "ok"}
```

### `app/database.py`

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import DeclarativeBase, sessionmaker

DATABASE_URL = "sqlite:///./db.sqlite3"

engine = create_engine(DATABASE_URL, connect_args={"check_same_thread": False})
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)


class Base(DeclarativeBase):
    pass


def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

### `tests/test_home.py`

```python
from fastapi.testclient import TestClient
from app.main import app, PROJECT_NAME

client = TestClient(app)


def test_home_returns_200_with_project_name():
    response = client.get("/")
    assert response.status_code == 200
    assert PROJECT_NAME in response.text


def test_health_ok():
    response = client.get("/health")
    assert response.status_code == 200
    assert response.json() == {"status": "ok"}
```

## How to test

    docker compose exec -T app pytest --cov=app --cov-report=term-missing --cov-fail-under=100

Covered:
- Every route — happy path + error path.
- Every Pydantic validator.
- Every model method with logic.
- CRUD functions / FastAPI dependencies.

Not covered:
- `app/__init__.py`.
- `app = FastAPI(...)` and `templates = Jinja2Templates(...)` constructors.
- Autogenerated Alembic.

## Do not bring in

- Django — if Django feels right, use `django-htmx` instead of this stack.
- Flask — FastAPI gives the same + async + typing.
- React, Vue — htmx is here precisely to avoid them.
- Tailwind — Pico.css classless.
- Celery — for background work, try FastAPI BackgroundTasks first.
- GraphQL — REST + htmx fragments is simpler.
- Separate DTOs per layer — Pydantic schemas in controllers, SQLAlchemy models for data is enough.
- Running Python on the host — everything through docker compose.
