## When to pick this stack

Pick this for a desktop app that runs offline, lives in the tray, touches local files, or sends native notifications: a notes app with full-text search over files, a clipboard manager, a local reader, a timer. Electron works, but weighs 3–5× more; Tauri uses the system webview and produces ~5–10 MB binaries. SvelteKit keeps the frontend small with minimal boilerplate.

## Minimal MVP tech

- Rust (stable toolchain via `rustup`)
- Node.js 22 LTS
- `@tauri-apps/cli@^2` — CLI and build
- `@tauri-apps/api@^2` — JS API for the Rust backend
- SvelteKit (frontend, SPA mode)
- `better-sqlite3` or `rusqlite` inside the Rust backend — local DB
- Tailwind or plain CSS — styling
- `vitest` for JS tests, `cargo test` for the Rust backend

## Phase 1 recipe (handoff)

SDD does not automate Tauri — a Rust toolchain and (on some OSes) system build tools are required. Claude builds Phase 1 from this recipe.

**What Phase 1 must produce:**

1. Initialize:

        npm create tauri-app@latest -- --template svelte-ts --manager npm

   Answers: name `<slug>`, frontend SvelteKit, package manager npm.
2. `npm install`.
3. Directory shape (the CLI generates most of this):

        src/              — SvelteKit frontend
        ├── routes/
        │   ├── +layout.svelte
        │   └── +page.svelte
        ├── lib/
        │   └── ipc.ts    — invoke(...) wrappers
        src-tauri/        — Rust backend
        ├── src/
        │   ├── main.rs
        │   └── commands.rs
        ├── Cargo.toml
        ├── tauri.conf.json
        └── icons/

4. `src/routes/+page.svelte` — home page with a heading (project name) and a placeholder for the first feature.
5. `src-tauri/src/commands.rs` — first command `#[tauri::command] fn health() -> &'static str { "ok" }`, registered in `main.rs`.
6. First tests:
   - `src/routes/+page.test.ts` — vitest, that the heading renders.
   - `src-tauri/src/commands.rs` — `#[test] fn health_returns_ok()`.

### Running

- Dev: `npm run tauri dev` — window opens, frontend hot-reloads.
- Build: `npm run tauri build` — produces `.dmg` / `.msi` / `.AppImage`.

### Host environment

- `rustup` + stable toolchain.
- macOS: `xcode-select --install` (for the linker).
- Linux: `libwebkit2gtk-4.1-dev`, `libayatana-appindicator3-dev`, `librsvg2-dev`, `build-essential`.
- Windows: Visual Studio Build Tools + WebView2 runtime.

`tauri info` inside the project reports what's missing.

## How to test

Two tracks — JS (frontend) and Rust (backend):

- JS: `npm test` (vitest), coverage `npm run test:coverage`. 100% on changed `src/**/*.{ts,svelte}`.
- Rust: `cd src-tauri && cargo test`, coverage via `cargo tarpaulin --out Html` (install `cargo install cargo-tarpaulin`). 100% on changed `.rs`.

Covered:
- Every Svelte page and component — at least one render.
- Every utility in `src/lib/*.ts`.
- Every `#[tauri::command]` on the Rust side.
- Every SQLite access function.

Not covered:
- `main.rs` (entry point).
- Autogenerated (Tauri builder configs, `+layout.svelte` without logic).

## Do not bring in

- Electron — the whole point of Tauri is that it's lighter.
- Angular, React — SvelteKit is simpler and smaller.
- Redux, MobX — Svelte runes / stores cover state.
- Heavy IPC frameworks — Tauri's `invoke()` is already simple.
- Auto-updater and code signing at MVP — defer to production deploy.
