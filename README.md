# SDD для начинающих

**SDD (Spec-Driven Development) для vibe coders** — набор из четырёх скиллов для Claude Code, которые превращают твою идею в MVP через одно длинное «интервью» и серию коротких строительных итераций. SDD сам подбирает подходящий стек под идею: обычное веб-приложение → Django + htmx; SPA с drag-and-drop → Next.js; мобильное → Flutter; CLI → Python + Typer; и так далее.

Всё на русском. Стек не обсуждается — ты рассказываешь про идею, SDD выбирает. Для самых частых стеков (Django + htmx и Next.js) Фаза 1 поднимается автоматически в Docker. Для остальных — SDD выдаёт готовый рецепт в `PROJECT.md`, и Claude строит Phase 1 по нему.

## Кому это подходит

Ты подходишь, если:
- У тебя есть идея MVP **для себя или узкого круга** (не для миллионов юзеров).
- Ты умеешь пользоваться терминалом на уровне «открыть папку, запустить команду».
- Ты готов не спорить со стеком — его выберет SDD, чтобы ты не тратил на это время.
- Ты ок с тем, что Claude делает бóльшую часть работы, а ты даёшь обратную связь.

Примеры идей, которые SDD закрывает:
- Веб-приложения: трекер книг/фильмов/тренировок, менеджер рецептов/бюджета/закладок, блог, чек-листы, внутренний инструмент. → SDD выберет Django + htmx и построит Фазу 1 автоматически.
- SPA с живым интерактивом: канбан с drag-and-drop, совместный редактор, таблица с инлайн-редактом. → SDD выберет Next.js и тоже построит Фазу 1 автоматически.
- Мобильное приложение, iOS/Android. → SDD выберет Flutter, выдаст готовый рецепт в `PROJECT.md`, и Claude построит Phase 1 по нему (scaffold вручную, потому что нужен Android SDK / Xcode на хосте).
- Desktop-приложение. → Tauri, та же схема handoff.
- CLI-утилита. → Python + Typer, handoff.
- Браузерная 2D-игра. → Phaser.js + Vite, handoff.

SDD не отказывается от идей. Если идея выходит за рамки веб-стеков, скилл просто подберёт стек, который подходит, и задокументирует рецепт в `PROJECT.md`.

## Что ты получишь

После прогона всех фаз проекта у тебя будет (детали зависят от выбранного стека):

- **Работающий MVP** — веб-приложение в Docker (одна команда `docker compose up`), или мобильный билд, или CLI-утилита в venv, или desktop-приложение. Конкретный способ запуска прописан в `PROJECT.md`.
- **Админка** — если стек это умеет из коробки (Django даёт её бесплатно; для Next.js и handoff-стеков — нет).
- **100% покрытие тестами** на всех файлах, которые мы меняли. Тесты запускаются командой из `PROJECT.md` (django test / vitest / pytest / flutter test — в зависимости от стека). Фаза не коммитится, пока покрытие не зелёное.
- **История в git** — один коммит на фазу (`phase 1: setup`, `phase 2: ...`). Любую фазу можно откатить командой `/sdd-undo`.
- **`PROJECT.md`** в корне — единый файл со спецификацией, стеком (с рецептом Фазы 1), и планом по фазам. Легко вернуться через месяц и понять, что ты делал.
- **`CLAUDE.md`** — короткий указатель на `PROJECT.md`, чтобы Claude в следующих сессиях не тащил чужой стек в твой проект.
- Последняя фаза в плане — всегда про деплой (на Render.com / TestFlight / GitHub Pages / PyPI — опять же по стеку).

## Стеки, из которых SDD выбирает

SDD читает идею и выбирает один стек. Ты не выбираешь — скилл выбирает, чтобы ты не тратил время на холивар. Ниже — поддерживаемые стеки и когда SDD их берёт:

| Стек | Когда SDD его выберет | Режим `/sdd-impl` |
|---|---|---|
| **Django 5 + htmx + SQLite + Pico.css + Docker** | Обычное веб-приложение: трекер, менеджер, блог, внутренний инструмент. Это дефолт. | scaffold — Фаза 1 автоматическая |
| **Next.js 15 + Prisma + SQLite + Tailwind + Docker** | SPA-интерактив: drag-and-drop, совместный редактор, инлайн-редакт таблиц. | scaffold — Фаза 1 автоматическая |
| **Flutter + Riverpod + sqflite** | Мобильное iOS/Android приложение. | handoff — SDD даёт рецепт, Claude строит вручную |
| **Tauri 2 + SvelteKit + SQLite** | Desktop (macOS/Windows/Linux). | handoff |
| **Python + Typer + Rich + pytest** | CLI-утилита. | handoff |
| **FastAPI + htmx + SQLite + SQLAlchemy + Docker** | Лёгкий веб-инструмент без админки и auth. | handoff |
| **Phaser 3 + Vite + TypeScript + Vitest** | Браузерная 2D-игра. | handoff |

**scaffold** — SDD сам создаст Dockerfile, compose, проект и все первые файлы, запустит контейнер, прогонит тесты и закоммитит `phase 1: ...`. Ты просто смотришь в браузер.

**handoff** — SDD напишет в `PROJECT.md` полный рецепт Фазы 1 (список файлов, команды, тесты) и сделает marker-коммит. Дальше запускаешь Claude в этой же папке — он строит по `PROJECT.md`. Handoff нужен там, где для сборки нужен тулчейн на хосте (Rust, Android SDK, Xcode, venv), который выходит за рамки «только Docker и git».

В обоих режимах `PROJECT.md` самодостаточен — там и план, и весь стек-специфичный рецепт.

## Что нужно поставить заранее

### 1. Docker Desktop

Качай с [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/) и установи. Запусти, подожди, пока иконка кита перестанет дёргаться.

На Linux вместо Docker Desktop подойдёт Docker Engine + docker-compose-plugin:
```bash
sudo apt install docker.io docker-compose-plugin   # Debian/Ubuntu
sudo dnf install docker docker-compose-plugin      # Fedora
```

### 2. Git

Почти наверняка уже есть. Проверка:
```bash
git --version
```

Если нет:
- macOS: `xcode-select --install`
- Linux: `sudo apt install git` или `sudo dnf install git`

### 3. Claude Code

Это инструмент, который запускает SDD-скиллы. Ставь по инструкции с [claude.com/claude-code](https://claude.com/claude-code).

### 4. Плагин `frontend-design`

Чтобы UI был не «дефолтно-AI-шный», SDD обращается к плагину `frontend-design`. Без него `/sdd-impl` откажется строить.

В Claude Code запусти:
```
/plugin marketplace add anthropics/claude-code
/plugin install frontend-design@claude-plugins-official
```

Потом перезапусти Claude Code.

### Поддержка ОС

**macOS и Linux.** Windows (нативный или WSL) не поддерживается — слишком много мелких проблем с путями и томами, которые будут отвлекать. Если ты на Windows — поставь виртуалку с Ubuntu или работай на Mac друга.

В терминале:

```bash
git clone <url-этого-репозитория>
cd SDD4beginners
./install.sh
```

Установщик скопирует команды на место, проверит окружение и, если чего-то не хватает (Docker, git, плагина), скажет что именно и как починить. Старые версии скиллов сохранятся в бэкапе.

После установки **перезапусти Claude Code**, чтобы он подхватил новые команды.

## Первый проект за 30 минут

Допустим, ты хочешь сделать трекер книг: заносить прочитанное, ставить оценки, писать короткие заметки.

### Шаг 1. Пустая папка

```bash
mkdir ~/projects/book-tracker
cd ~/projects/book-tracker
```

### Шаг 2. `/sdd-idea` — спроектировать

Открой Claude Code в этой папке, скажи:

> `/sdd-idea` Хочу приложение для учёта прочитанных книг: оценка, короткая заметка, статистика по месяцам.

Claude начнёт **интервью на 8–12 вопросов**: кто пользователи, как выглядит типичная сессия, какие данные хранятся, логин ли нужен, на что похоже визуально, какие 3 фичи самые важные. Отвечай как есть — ты лучше знаешь, что тебе нужно.

После интервью Claude скажет один раз, какой стек он выбрал (в случае трекера книг это будет Django + htmx) — это не вопрос, а констатация, обсуждать стек не надо.

Потом Claude предложит MVP — список того, что имеет смысл собрать первым. Если ок — подтверди; если нет — скажи, что убрать/добавить.

После одобрения в папке появится **`PROJECT.md`** — спека + стек с рецептом Фазы 1 + список из ~5 фаз.

Claude спросит: продолжить интервью, поменять план, или поехали строить. Если нужен ещё раунд вопросов — просто скажи «продолжим интервью»; если хочешь поменять фазы местами или вырезать что-то — «поменять план». Когда всё нравится — «поехали, строим».

### Шаг 3. `/sdd-impl` — Фаза 1

В терминале (или той же сессии Claude Code):

> `/sdd-impl`

Claude сам развернёт проект по плану из `PROJECT.md`: создаст все нужные файлы, поднимет приложение, напишет первые тесты, пересоберёт с фиксами, если что-то нужно улучшить, и закоммитит всё как `phase 1`. В конце получишь **ссылку на работающее приложение** и откроешь его в браузере.

Для стеков, которые SDD автоматически не собирает (мобильные, desktop, CLI, игры), `/sdd-impl` покажет, что строить дальше — план уже готов в `PROJECT.md`, а Claude построит Фазу 1 по нему в обычной сессии.

### Шаг 4. `/sdd-impl` — следующие фазы

Каждая фаза добавляет **одну вертикальную фичу** (не «все модели сначала, потом все вьюхи» — а «эта фича целиком: модель + вьюха + шаблон + тесты»).

> `/sdd-impl`

Например, Фаза 2 может быть «добавление книги»: модель `Book`, форма, список на главной, тесты. После фазы — новый коммит, приложение работает, ты можешь руками добавить книгу в браузере.

Повторяй `/sdd-impl`, пока все фазы не станут `[x]`. Последняя фаза — всегда про деплой на Render.com.

### Шаг 5. Что-то пошло не так?

Фаза получилась кривой — страница некрасивая, логика странная, тесты ставятся формально:

> `/sdd-undo`

Claude откатит последнюю фазу через `git revert` (история не перепишется — это безопасно). Задачи фазы в `PROJECT.md` снова станут `[ ]`. Дальше — либо сразу `/sdd-impl`, чтобы попробовать заново, либо подправь `PROJECT.md` руками (например, перепиши чекпоинт фазы) и потом `/sdd-impl`.

### Шаг 6. Хочу добавить что-то новое

Проект уже работает, и ты хочешь новую фичу — например, делиться книгами с другом через публичную ссылку:

> `/sdd-feature` Хочу, чтобы можно было делиться любой книгой через публичную ссылку, без логина.

Claude задаст 2–3 вопроса (где в UI, как выглядит публичная страница, нужен ли срок жизни ссылки), и допишет новые фазы в `PROJECT.md`. Предыдущий план бэкапится в `PROJECT.v<N>.md`. Дальше — `/sdd-impl`, как обычно.

## Четыре команды — справочник

| Команда | Что делает | Когда запускать |
|---|---|---|
| `/sdd-idea` | Интервью → `PROJECT.md`. Никакого кода. | В пустой папке, когда есть идея. |
| `/sdd-impl` | Строит следующую невыполненную фазу. Коммит в конце. | Когда есть `PROJECT.md`. Запускай по одной фазе за раз. |
| `/sdd-undo` | Откатывает последнюю фазу через `git revert`. | Когда фаза получилась плохо. |
| `/sdd-feature` | Интервью → новые фазы в `PROJECT.md`. Кода не пишет. | Когда проект уже есть и нужна новая фича. |

Триггеры срабатывают и на русском, и на английском. Не обязательно писать именно `/sdd-idea` — достаточно «давай спроектируем приложение» или «new project», и Claude подхватит скилл сам.

## Как устроен проект

Структура папки и повседневные команды (запуск, тесты, логи) зависят от выбранного стека. После Фазы 1 всё нужное — пути, команды, правила — лежит в `PROJECT.md` и коротком `CLAUDE.md` в корне твоего проекта. Заглядываешь туда один раз и видишь всё.

## На чём держится весь подход

- **`PROJECT.md` — источник правды.** Спека, стек, план, галочки по задачам — в одном файле. Вернёшься через месяц — Claude откроет его и всё вспомнит.
- **Один коммит на фазу.** История git — это твой безопасный «undo». `/sdd-undo` делает revert-коммит (не переписывает историю), можно запускать подряд.
- **100% покрытия на изменённых файлах.** Без тестов следующая фаза незаметно ломает предыдущую; с тестами — не может. Фаза не коммитится, пока тесты не зелёные.

## FAQ и типичные проблемы

### Claude не видит мои команды `/sdd-*`

- Перезапусти Claude Code после установки.
- Проверь, что папки появились: `ls ~/.claude/skills/sdd-*`. Если нет — запусти `./install.sh` из папки репозитория.

### Нужен плагин `frontend-design`

В Claude Code:
```
/plugin marketplace add anthropics/claude-code
/plugin install frontend-design@claude-plugins-official
```
Перезапусти Claude Code и попробуй снова.

### Docker не запущен

- macOS: `open -a Docker`, подожди 10–15 секунд.
- Linux: `sudo systemctl start docker`.
- Если только что поставил Docker первый раз — возможно, нужна перезагрузка.

### `/sdd-impl` говорит «сначала нужен `/sdd-idea`»

У тебя в папке нет `PROJECT.md`. Запусти `/sdd-idea` и спроектируй проект.

### Фаза получилась плохо, но я уже закоммитил

`/sdd-undo`. Это безопасно: `git revert`, а не `reset`. Можно запускать несколько раз подряд, чтобы откатиться на несколько фаз назад.

### `PROJECT.md` выглядит странно, хочу его править руками

Можно. Это обычный Markdown. Не трогай задачи в **уже завершённых фазах** (`- [x]`) — они представляют реальный код в git. В будущих фазах можешь менять что угодно. Перед большой правкой сделай бэкап: `cp PROJECT.md PROJECT.v-manual.md`.

## Обновление и удаление

### Обновить SDD

```bash
cd SDD4beginners
git pull
./install.sh
```

Старые версии скиллов уезжают в `~/.claude/skills/sdd-*.bak-<дата>` — ничего не пропадёт.

### Удалить SDD

```bash
rm -rf ~/.claude/skills/sdd-idea ~/.claude/skills/sdd-impl ~/.claude/skills/sdd-undo ~/.claude/skills/sdd-feature
# (на всякий случай — если у тебя осталась старая копия скрипта-доктора)
rm -f ~/.claude/scripts/sdd-doctor.sh
```

Проекты, которые ты уже построил, никак не пострадают — они самодостаточны (весь план и рецепт лежит в `PROJECT.md`).

## Коротко

1. Поставь Docker, git, Claude Code, плагин `frontend-design`.
2. `git clone` этого репо, `./install.sh`.
3. В пустой папке — `/sdd-idea`, ответь на вопросы. SDD сам выберет подходящий стек под твою идею.
4. `/sdd-impl` по одной фазе, пока MVP не готов (для scaffold-стеков — автоматически; для handoff — Claude строит по рецепту в `PROJECT.md`).
5. `/sdd-undo` если что-то пошло не так. `/sdd-feature` когда хочется нового.

Погнали.
